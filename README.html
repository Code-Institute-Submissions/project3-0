Magento Ecommerce system

Magento is an open-source and commercial e-commerce platform written in PHP.

Magento uses the MySQL or MariaDB relational database management system, the PHP programming language, and elements of the Zend Framework.

It applies the conventions of object-oriented programming and model–view–controller architecture. Magento also uses the entity–attribute–value model to store data. On top of that, Magento 2 introduced the Model-View-ViewModel pattern to its front-end code using the JavaScript library Knockout.js.

The current opensourtce version is Magento 2.3.1 and this is the version i will be working with. Sometimes I will refer to it simply as M2

Magento API description and summary

The Magento web API framework provides integrators and developers the means to use web services to communicate with the Magento ecommerce system.

The Magento API support both REST (Representational State Transfer) and SOAP (Simple Object Access Protocol). for my project I will use the REST API but the Magento 2 web API coverage is the same for both REST and SOAP. The documentation for the Mahento API is locatd here https://devdocs.magento.com/guides/v2.3/get-started/bk-get-started-api.html The learning outcomes for this section of the course include: being able to access external resources, work with external data, deserialize JSON data, and create data tables with pagination.

Project outline

The project will create a Html/Javascript/JQuery file that will connect to the Magento 2 API on my domain.
The page will return product, customer or order data based on the requests made from the user.

Project phases.

The first objective is to get the connection working to the M2 API.
A basic html file will be created based on this mockup to test the connection.
The html file will have three sections where the user will be able to enter the search string and see the returned data
  1. order id  - to return order information   
  2. product type - to return matching product information
  3. customer name  to return matching customer informatio
Simple styling of the data will be done initially and then the code will be integrated into a bootstrap template.
The page should react when the customer enters the order id , product or customer info and  displays some relevant data
